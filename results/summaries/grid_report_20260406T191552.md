## Aggregation Semantics

- **Timing fields** (prefill_ms, decode_ms, total_ms, tok/s): median across repeats.
- **Memory and phase fields** (peak_allocated, generation_overhead, peak_phase,
  runtime_kv_ratio, coarse_kv_ratio): taken from the **median-time representative**
  run — the repeat whose total_time_ms is closest to the median. Memory allocation
  is deterministic for a given sequence length, but selecting the median-time run
  avoids outlier interactions from timing anomalies.

## Measurement Semantics

- **Theoretical KV cache**: formula-based (`2 * layers * batch * seq * kv_heads * head_dim * 2`), never from CUDA.
- **Observed peak allocated**: `torch.cuda.max_memory_allocated()` during generation.
- **Overhead above weights** (coarse audit): peak_allocated - theoretical_weight_memory.
  Includes load-time overhead (allocator pools, non-weight buffers).
- **Generation overhead above load** (primary runtime metric): peak_allocated - memory_after_load.
  Isolates memory growth caused by generation (KV cache, activations, attention matrices, logits, temporaries).
- **Peak phase**: whether `max_memory_allocated` was reached during prefill or decode,
  inferred from memory snapshots taken after each phase.

**Neither ratio is a direct component-wise decomposition.** Both are consistency
checks comparing a formula-based estimate against a CUDA-observed aggregate.

## Model-Specific Observations

### progen2-small

- Configurations tested: 25
- Peak phase: 0 prefill, 25 decode, 0 unclear
- Runtime KV ratio (TheoKV / GenOverhead):
  - Short sequences (seq <= 128): avg 33.8%
  - Long sequences (seq >= 400): avg 30.4%
  - Overall average: 32.1%

Theoretical KV remains a minority of runtime growth across the tested range.
A substantial non-KV runtime baseline persists even after separating post-load
growth. This baseline is consistent with allocator reserved pools, activation
buffers, and logit computation overhead that scale weakly with sequence length.
KV cache grows linearly but does not dominate the overhead budget at any
tested sequence length.

### protgpt2

- Configurations tested: 25
- Peak phase: 6 prefill, 19 decode, 0 unclear
- Runtime KV ratio (TheoKV / GenOverhead):
  - Short sequences (seq <= 128): avg 98.3%
  - Long sequences (seq >= 400): avg 92.1%
  - Overall average: 94.2%

Theoretical KV becomes large relative to generation-overhead-above-load
at longer sequences. However, peak attribution depends on the prompt/decode
split: 6 configs peaked during prefill and 19
peaked during decode. Configurations with the same final sequence length but
different prompt/decode ratios can exhibit different peak phase behavior.
This suggests that both prefill-time activations and decode-time KV cache
growth contribute to peak memory, with the balance shifting based on the
prompt/decode split.

## Combined Pattern (secondary note)

These observations are consistent with materially different memory behavior
between the two models, driven primarily by architectural differences
(36 layers vs 12 layers, resulting in 3.75x more KV cache per token for ProtGPT2).
Neither model's behavior can be fully attributed to a single component
from these measurements alone.
